\subsubsection[{T\+N\+\_\+\+Tick\+Cnt}]{\setlength{\rightskip}{0pt plus 5cm}typedef unsigned long {\bf T\+N\+\_\+\+Tick\+Cnt}}}\label{tn__common_8h_ac885b63d00c063de61cdbd80bf26d8aa}


Type for system tick count, it is used by the kernel to represent absolute tick count value as well as relative timeouts. 

When it is used as a timeout value, it represents the maximum number of system ticks to wait.

Assume user called some system function, and it can't perform its job immediately (say, it needs to lock mutex but it is already locked, etc).

So, function can wait or return an error. There are possible {\ttfamily timeout} values and appropriate behavior of the function\+:


\begin{DoxyItemize}
\item {\ttfamily timeout} is set to {\ttfamily 0}\+: function doesn't wait at all, no context switch is performed, {\ttfamily \hyperlink{tn__common_8h_aa43bd3da1ad4c1e61224b5f23b369876a5b4d73fde6b5d1c9579c02e6aafce1fb}{T\+N\+\_\+\+R\+C\+\_\+\+T\+I\+M\+E\+O\+U\+T}} is returned immediately.
\item {\ttfamily timeout} is set to {\ttfamily \hyperlink{tn__arch__example_8h_a16909a2a05bc2bc628109d843208a76b}{T\+N\+\_\+\+W\+A\+I\+T\+\_\+\+I\+N\+F\+I\+N\+I\+T\+E}}\+: function waits until it eventually {\bfseries can} perform its job. Timeout is not taken in account, so {\ttfamily \hyperlink{tn__common_8h_aa43bd3da1ad4c1e61224b5f23b369876a5b4d73fde6b5d1c9579c02e6aafce1fb}{T\+N\+\_\+\+R\+C\+\_\+\+T\+I\+M\+E\+O\+U\+T}} is never returned.
\item {\ttfamily timeout} is set to other value\+: function waits at most specified number of system ticks. Strictly speaking, it waits from {\ttfamily (timeout -\/ 1)} to {\ttfamily timeout} ticks. So, if you specify that timeout is 1, be aware that it might actually don't wait at all\+: if {\itshape \hyperlink{time_ticks}{system timer}} interrupt happens just while function is putting task to wait (with interrupts disabled), then I\+S\+R will be executed right after function puts task to wait. Then {\ttfamily \hyperlink{tn__sys_8h_a944d96c7a5d442d271115b6cb22a085b}{tn\+\_\+tick\+\_\+int\+\_\+processing()}} will immediately remove the task from wait queue and make it runnable again.

So, to guarantee that task waits {\itshape at least} 1 system tick, you should specify timeout value of {\ttfamily 2}.
\end{DoxyItemize}

{\bfseries Note} also that there are other possible ways to make task runnable\+:


\begin{DoxyItemize}
\item if task waits because of call to {\ttfamily \hyperlink{tn__tasks_8h_ae768a72ca0efde5767796cc1770bd45e}{tn\+\_\+task\+\_\+sleep()}}, it may be woken up by some other task, by means of {\ttfamily \hyperlink{tn__tasks_8h_abb88bc8b1cec6b82e7b6e2e90d0e155a}{tn\+\_\+task\+\_\+wakeup()}}. In this case, {\ttfamily \hyperlink{tn__tasks_8h_ae768a72ca0efde5767796cc1770bd45e}{tn\+\_\+task\+\_\+sleep()}} returns {\ttfamily \hyperlink{tn__common_8h_aa43bd3da1ad4c1e61224b5f23b369876afb291924237186f5765865256c75e639}{T\+N\+\_\+\+R\+C\+\_\+\+O\+K}}.
\item independently of the wait reason, task may be released from wait forcibly, by means of {\ttfamily \hyperlink{tn__tasks_8h_ae90ecdbd7d87d050c2c98ec775e6efc7}{tn\+\_\+task\+\_\+release\+\_\+wait()}}. It this case, {\ttfamily \hyperlink{tn__common_8h_aa43bd3da1ad4c1e61224b5f23b369876ae5cbacb0fb01805c9003046c08bf4356}{T\+N\+\_\+\+R\+C\+\_\+\+F\+O\+R\+C\+E\+D}} is returned by the waiting function. (the usage of the {\ttfamily \hyperlink{tn__tasks_8h_ae90ecdbd7d87d050c2c98ec775e6efc7}{tn\+\_\+task\+\_\+release\+\_\+wait()}} function is discouraged though) 
\end{DoxyItemize}
