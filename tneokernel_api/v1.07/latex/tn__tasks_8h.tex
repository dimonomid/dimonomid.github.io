\subsection{Detailed Description}
\hypertarget{tn__tasks_8h_tn_tasks__tasks}{}\subsection{Task}\label{tn__tasks_8h_tn_tasks__tasks}
In T\+Neo, a task is a branch of code that runs concurrently with other tasks from the programmer's point of view. Indeed, tasks are actually executed using processor time sharing. Each task can be considered to be an independed program, which executes in its own context (processor registers, stack pointer, etc.).

Actually, the term {\itshape thread} is more accurate than {\itshape task}, but the term {\itshape task} historically was used in T\+N\+Kernel, so T\+Neo keeps this convention.

When kernel decides that it's time to run another task, it performs {\itshape context switch}\+: current context (at least, values of all registers) gets saved to the preempted task's stack, pointer to currently running task is altered as well as stack pointer, and context gets restored from the stack of newly running task.\hypertarget{tn__tasks_8h_tn_tasks__states}{}\subsection{Task states}\label{tn__tasks_8h_tn_tasks__states}
For list of task states and their description, refer to {\ttfamily enum \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6}{T\+N\+\_\+\+Task\+State}}.\hypertarget{tn__tasks_8h_tn_tasks__creating}{}\subsection{Creating/starting tasks}\label{tn__tasks_8h_tn_tasks__creating}
Create task and start task are two separate actions; although you can perform both of them in one step by passing {\ttfamily \hyperlink{tn__tasks_8h_a8fa2ef577d6bd159b3fae559839f98d5a0c9352496e4465eb7e1b29dab7544acc}{T\+N\+\_\+\+T\+A\+S\+K\+\_\+\+C\+R\+E\+A\+T\+E\+\_\+\+O\+P\+T\+\_\+\+S\+T\+A\+R\+T}} flag to the {\ttfamily \hyperlink{tn__tasks_8h_a548d5adda09d1b4e393b5df0e9e1a7a5}{tn\+\_\+task\+\_\+create()}} function.\hypertarget{tn__tasks_8h_tn_tasks__stopping}{}\subsection{Stopping/deleting tasks}\label{tn__tasks_8h_tn_tasks__stopping}
Stop task and delete task are two separate actions. If task was just stopped but not deleted, it can be just restarted again by calling {\ttfamily \hyperlink{tn__tasks_8h_a3c14832fe7dfd2c44678608767f40230}{tn\+\_\+task\+\_\+activate()}}. If task was deleted, it can't be just activated\+: it should be re-\/created by {\ttfamily \hyperlink{tn__tasks_8h_a548d5adda09d1b4e393b5df0e9e1a7a5}{tn\+\_\+task\+\_\+create()}} first.

Task stops execution when\+:


\begin{DoxyItemize}
\item it calls {\ttfamily \hyperlink{tn__tasks_8h_a23e562731e616f01e325bc9830bcc021}{tn\+\_\+task\+\_\+exit()}};
\item it returns from its task body function (it is the equivalent to {\ttfamily tn\+\_\+task\+\_\+exit(0)})
\item some other task calls {\ttfamily \hyperlink{tn__tasks_8h_a8ae6615de7022a327bdcd4c37a0f5b90}{tn\+\_\+task\+\_\+terminate()}} passing appropriate pointer to {\ttfamily struct \hyperlink{structTN__Task}{T\+N\+\_\+\+Task}}.
\end{DoxyItemize}\hypertarget{tn__tasks_8h_tn_tasks__scheduling}{}\subsection{Scheduling rules}\label{tn__tasks_8h_tn_tasks__scheduling}
T\+Neo always runs the most privileged task in state \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6a02783ac7808aeda318a6f506b7a276dc}{{\ttfamily R\+U\+N\+N\+A\+B\+L\+E}}. In no circumstances can task run while there is at least one task is in the \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6a02783ac7808aeda318a6f506b7a276dc}{{\ttfamily R\+U\+N\+N\+A\+B\+L\+E}} state with higher priority. Task will run until\+:


\begin{DoxyItemize}
\item It becomes non-\/runnable (say, it may wait for something, etc)
\item Some other task with higher priority becomes runnable.
\end{DoxyItemize}

Tasks with the same priority may be scheduled in round robin fashion by getting a predetermined time slice for each task with this priority. Time slice is set separately for each priority. By default, round robin is turned off for all priorities.\hypertarget{tn__tasks_8h_tn_tasks__idle}{}\subsection{Idle task}\label{tn__tasks_8h_tn_tasks__idle}
T\+Neo has one system task\+: an idle task, which has lowest priority. It is always in the state \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6a02783ac7808aeda318a6f506b7a276dc}{{\ttfamily R\+U\+N\+N\+A\+B\+L\+E}}, and it runs only when there are no other runnable tasks.

User can provide a callback function to be called from idle task, see \hyperlink{tn__sys_8h_ac53892c962d429fadf7cf508024eec66}{T\+N\+\_\+\+C\+B\+Idle}. It is useful to bring the processor to some kind of real idle state, so that device draws less current.

This function terminates the currently running task. 

The task is moved to the \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6a46ec35f78d82d1b14f1dd35b3e762b3b}{{\ttfamily D\+O\+R\+M\+A\+N\+T}} state.

After exiting, the task may be either deleted by the {\ttfamily \hyperlink{tn__tasks_8h_adbc4dbbd4a57b73642b76550a0c7c83f}{tn\+\_\+task\+\_\+delete()}} function call or reactivated by the {\ttfamily \hyperlink{tn__tasks_8h_a3c14832fe7dfd2c44678608767f40230}{tn\+\_\+task\+\_\+activate()}} / {\ttfamily \hyperlink{tn__tasks_8h_a46c30078ae437c75121965f70c9d7b4c}{tn\+\_\+task\+\_\+iactivate()}} function call. In this case task starts execution from beginning (as after creation/activation). The task will have the lowest precedence among all tasks with the same priority in the \hyperlink{tn__tasks_8h_a5e12e8a0ab280b515f44bf3fee1210a6a02783ac7808aeda318a6f506b7a276dc}{{\ttfamily R\+U\+N\+N\+A\+B\+L\+E}} state.
