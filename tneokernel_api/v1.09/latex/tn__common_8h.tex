{\footnotesize\ttfamily typedef unsigned long \mbox{\hyperlink{tn__common_8h_ac885b63d00c063de61cdbd80bf26d8aa}{T\+N\+\_\+\+Tick\+Cnt}}}



Type for system tick count, it is used by the kernel to represent absolute tick count value as well as relative timeouts. 

When it is used as a timeout value, it represents the maximum number of system ticks to wait.

Assume user called some system function, and it can\textquotesingle{}t perform its job immediately (say, it needs to lock mutex but it is already locked, etc).

So, function can wait or return an error. There are possible {\ttfamily timeout} values and appropriate behavior of the function\+:


\begin{DoxyItemize}
\item {\ttfamily timeout} is set to {\ttfamily 0}\+: function doesn\textquotesingle{}t wait at all, no context switch is performed, {\ttfamily \#T\+N\+\_\+\+R\+C\+\_\+\+T\+I\+M\+E\+O\+UT} is returned immediately.
\item {\ttfamily timeout} is set to {\ttfamily \#T\+N\+\_\+\+W\+A\+I\+T\+\_\+\+I\+N\+F\+I\+N\+I\+TE}\+: function waits until it eventually {\bfseries{can}} perform its job. Timeout is not taken in account, so {\ttfamily \#T\+N\+\_\+\+R\+C\+\_\+\+T\+I\+M\+E\+O\+UT} is never returned.
\item {\ttfamily timeout} is set to other value\+: function waits at most specified number of system ticks. Strictly speaking, it waits from {\ttfamily (timeout -\/ 1)} to {\ttfamily timeout} ticks. So, if you specify that timeout is 1, be aware that it might actually don\textquotesingle{}t wait at all\+: if {\itshape \mbox{\hyperlink{time_ticks}{system timer}}} interrupt happens just while function is putting task to wait (with interrupts disabled), then I\+SR will be executed right after function puts task to wait. Then {\ttfamily \mbox{\hyperlink{tn__sys_8h_aa44d297639e0520420890ef2bb7e2c1c}{tn\+\_\+tick\+\_\+int\+\_\+processing()}}} will immediately remove the task from wait queue and make it runnable again.

So, to guarantee that task waits {\itshape at least} 1 system tick, you should specify timeout value of {\ttfamily 2}.
\end{DoxyItemize}

{\bfseries{Note}} also that there are other possible ways to make task runnable\+:


\begin{DoxyItemize}
\item if task waits because of call to {\ttfamily \mbox{\hyperlink{tn__tasks_8h_ae768a72ca0efde5767796cc1770bd45e}{tn\+\_\+task\+\_\+sleep()}}}, it may be woken up by some other task, by means of {\ttfamily \mbox{\hyperlink{tn__tasks_8h_abb88bc8b1cec6b82e7b6e2e90d0e155a}{tn\+\_\+task\+\_\+wakeup()}}}. In this case, {\ttfamily \mbox{\hyperlink{tn__tasks_8h_ae768a72ca0efde5767796cc1770bd45e}{tn\+\_\+task\+\_\+sleep()}}} returns {\ttfamily \#T\+N\+\_\+\+R\+C\+\_\+\+OK}. ~\newline

\item independently of the wait reason, task may be released from wait forcibly, by means of {\ttfamily \mbox{\hyperlink{tn__tasks_8h_ae90ecdbd7d87d050c2c98ec775e6efc7}{tn\+\_\+task\+\_\+release\+\_\+wait()}}}. It this case, {\ttfamily \#T\+N\+\_\+\+R\+C\+\_\+\+F\+O\+R\+C\+ED} is returned by the waiting function. (the usage of the {\ttfamily \mbox{\hyperlink{tn__tasks_8h_ae90ecdbd7d87d050c2c98ec775e6efc7}{tn\+\_\+task\+\_\+release\+\_\+wait()}}} function is discouraged though) 
\end{DoxyItemize}
